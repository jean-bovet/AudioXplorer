## AudioXplorer 1.4.1

Bug fix release.

- **Real-time capture now works in the installed app.** The 1.4 release build was signed without the `com.apple.security.device.audio-input` entitlement, so macOS silently refused to prompt for microphone access — the real-time window opened but recorded silence.
- **Enable toggle in the real-time window no longer gets stuck on.** A type-truncation bug made `isRecording` always report `YES` after pause, so the toggle kept flipping itself back to ON.
- **Microphone permission is no longer re-requested on every launch.** The launch-time TCC trigger now runs only when authorization status is undetermined, instead of every time.

*If you previously installed 1.4 and granted microphone access, you may need to remove and re-add AudioXplorer in System Settings → Privacy & Security → Microphone — the new build's signature differs from 1.4's.*
